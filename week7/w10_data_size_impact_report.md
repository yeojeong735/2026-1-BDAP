# 웹 로그 데이터 사이즈 영향 분석 보고서 (Data Size Impact Report)

본 보고서는 CSIC 2010 웹 로그 데이터셋을 활용하여, 데이터 샘플링 크기가 탐색적 데이터 분석(EDA) 결과에 미치는 영향을 비교 분석한 결과입니다.

## 1. 실험 개요
- **목적**: 데이터 크기(10,000건 vs 61,065건)에 따른 주요 보안 지표 및 데이터 특성의 변화를 확인하고 분석의 신뢰성을 평가함.
- **분석 대상**:
    - **Baseline**: 10,000건 샘플링 데이터 (수업용 샘플)
    - **Experiment B**: 61,065건 전체 데이터 (Kaggle 원본 전량)

## 2. 주요 지표 비교 결과

| 분석 항목 | Baseline (10,000건) | Full Data (61,065건) | 차이 및 특이사항 |
| :--- | :---: | :---: | :--- |
| **총 데이터 수** | 10,000건 | 61,065건 | 약 6.1배 차이 |
| **정상 요청 (Normal)** | 5,895건 (59.0%) | 36,000건 (59.0%) | 비율의 일관성이 매우 높음 |
| **공격 요청 (Anomalous)** | 4,105건 (41.0%) | 25,065건 (41.0%) | 샘플링 후에도 공격 특성이 잘 유지됨 |
| **평균 URL 길이** | 61.54자 | 90.33자 | 데이터가 커질수록 평균 길이가 다소 길어짐 |
| **최대 URL 길이** | 865.0자 | 895자 | 전체 데이터에서 더 긴 비정상 요청 존재 |
| **GET 메서드 비율** | 69.8% | 70.6% | 메서드 분포의 안정성 확인 |
| **POST 메서드 비율** | 29.4% | 28.8% | 동일한 서비스 구조 반영 |

## 4. 분석 결과 및 해석

### 1) 샘플링의 대표성 (Representativeness)
- 10,000건의 샘플링 데이터는 전체 데이터(61,065건)와 비교했을 때 **라벨 분포(Normal/Anomalous)와 메서드 분포(GET/POST)에서 거의 차이가 없음**을 확인했습니다. 이는 무작위 샘플링이 모집단의 통계적 특성을 잘 보존하고 있음을 의미합니다.

### 2) 데이터 크기와 이상치(Outliers)
- **최대 URL 길이** 지표를 보면, 전체 데이터셋에서 10,000건 샘플에는 포함되지 않았던 훨씬 긴 URL(865자)이 발견되었습니다. 이는 보안 데이터 분석 시 **데이터의 양이 많을수록 탐지하기 어려운 변칙적 공격(Anomaly)이나 극단적인 케이스를 포착할 확률이 높아짐**을 보여줍니다.

### 3) 공격 키워드 탐지의 정밀도
- `'`나 `SELECT`와 같은 공격 키워드의 검출 빈도는 데이터 크기가 약 6배 증가함에 따라 거의 정비례하게 증가했습니다. 하지만 `UNION`과 같이 매우 드물게 나타나는 키워드는 소규모 샘플링(10,000건)에서는 아예 발견되지 않을 수도 있습니다(실제 결과 0건 vs 1건). 이는 **희귀한 공격 유형을 학습하기 위해서는 대용량 데이터셋이 필수적**임을 시사합니다.

### 4) 결론 및 제언
- 초기 분석 및 대시보드 구현 단계에서는 10,000건의 데이터로도 충분한 통찰을 얻을 수 있습니다.
- 그러나 실제 운영 환경에서 높은 정확도와 낮은 미탐율(False Negative)을 가진 머신러닝 모델을 구축하기 위해서는 **전체 데이터(61,065건)를 사용하여 미세한 공격 패턴까지 학습**시키는 과정이 반드시 필요합니다.
